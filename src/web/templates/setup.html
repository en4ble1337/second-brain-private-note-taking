iOS Shortcut Setup

  1. Open the Shortcuts app on your iPhone.
  2. Create a new shortcut.
  3. Add action: Record with Voice Memos.
  4. Add action: Get Contents of URL:
    • URL: {{ ingest_url }}
    • Method: POST
    • Headers: Authorization: Bearer <your-full-token>
    • Request Body: Form
    • Add field audio = Shortcut Input
    • Add field source = ios
  5. Assign to Action Button in Settings → Action Button.

Android Tasker Setup

  1. Create a Tasker Profile triggered by a Quick Settings tile.
  2. Add a task: Record Audio to a temp file.
  3. Add a task: HTTP Request:
    • Method: POST
    • URL: {{ ingest_url }}
    • Body: multipart with field audio = the recorded file, source = android
    • Header: Authorization: Bearer <your-full-token>

Remote Access

By default this appliance is only reachable on your home LAN. If you record a note while away from home, the ingest request will fail. Choose an option below based on your needs.

Local Only Default

No action needed. The brain is only accessible on your home network. Record notes at home; use Voice Memos as a manual buffer when away.

Tailscale Recommended

Creates a private encrypted network between your phone and this device. Works anywhere with an internet connection. Requires a free Tailscale account (their coordination server handles key exchange but never sees your content).

  1. Install Tailscale on this device: curl -fsSL https://tailscale.com/install.sh | sh && sudo tailscale up
  2. Install the Tailscale app on your iPhone/Android and sign in with the same account.
  3. Find this device's Tailscale IP: tailscale ip -4
  4. Update your Shortcut/Tasker URL to use that IP instead of mybrain.local.

Cloudflare Tunnel

Exposes this device via Cloudflare's infrastructure — no port forwarding, no VPN app on your phone. Free tier is sufficient. Note: Cloudflare acts as a TLS termination proxy, meaning your audio passes through their network.

  1. Create a free account at cloudflare.com and add a domain.
  2. Install cloudflared: curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64 -o /usr/local/bin/cloudflared && chmod +x /usr/local/bin/cloudflared
  3. Authenticate: cloudflared tunnel login
  4. Create a tunnel: cloudflared tunnel create brain
  5. Configure it to forward to http://localhost:80 and run as a service.
  6. Update your Shortcut/Tasker URL to use your Cloudflare tunnel hostname.

Privacy note: If this appliance's purpose is fully local and private, prefer Tailscale or stay LAN-only.
